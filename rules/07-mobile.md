# Mobile (Expo)

## A. Folder layout mirrors web

```
packages/mobile/
  app/                 # Expo Router file-based
  src/
    features/<feature>/  # components/, hooks.ts, types.ts
    lib/                 # trpc client, supabase client, env
    components/          # primitives
```

Features never import features.

## B. Storage discipline (load-bearing security rule)

- **SecureStore (`expo-secure-store`)** — tokens, JWT, refresh tokens, sensitive data.
- **AsyncStorage** — non-sensitive UI cache only.
- **MMKV** — optional perf upgrade later; persister adapter when used with TanStack Query.

Hook: asyncstorage-token-keys advisory — fires on `AsyncStorage.setItem` with keys matching `/token|auth|jwt|secret|password|refresh/i`.

## C. Animations + gestures

**Reanimated v3** is the only animation library. **`react-native-gesture-handler`** for gestures. Hook: rn-animated advisory — fires on `import { Animated } from 'react-native'`.

## D. Images

`expo-image` only. Hook-paired with C. Raw `<Image>` from `react-native` is allowed only for tiny static assets where caching is irrelevant.

## E. Platform branching

One-liners inline: `Platform.OS === 'ios' ? a : b`. More than one line of platform-specific code → sibling `Component.ios.tsx` / `Component.android.tsx` (Metro resolves automatically). Banned: deep `if (Platform.OS) { 20 lines }` blocks.

## F. State, forms, AI

Inherited from web rules (`06-frontend-web.md`) and `05-ai-agents.md`.

## G. Supabase auth

Supabase client configured with `expo-secure-store` adapter for token persistence. JWT refresh handled by the SDK. JWT propagation rule (`02-database-rls.md` #2) applies to Worker requests originating from mobile.

## H. OTA vs native rebuild

Native rebuild via EAS Build (not OTA via EAS Update) is required for changes to:

- `app.config.ts`
- `package.json` with native deps (post-install scripts, `react-native` peer, `ios/`/`android/` folders)
- `assets/` icons / splash
- Expo config plugins

Pre-deploy checklist lives in the project `RUNBOOK.md` (generated by `/scaffold-project`).

## I. Native-module package gate

`pnpm add` under `packages/mobile/` triggers a pause: check whether the target has native deps (post-install, `react-native` peer, `ios/`/`android/`). Also enforce Expo SDK compatibility. Hook: mobile-native-add advisory.

## J. `app.config.ts` is a function

Reads `process.env.APP_ENV` (`dev` | `preview` | `production`) and returns per-env values for `name`, `slug`, `ios.bundleIdentifier`, `android.package`, `extra.{supabaseUrl,workerUrl}`, `runtimeVersion`. No hardcoded strings.
