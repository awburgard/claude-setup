# /scaffold-project

Spins up a new monorepo project with the default stack and conventions pre-wired.

## Inputs

- Project name (kebab-case).
- Flags: `--web`, `--worker`, `--mobile` (defaults: `--worker --web`).
- Optional `--target-dir <path>` (default: `~/projects/<name>`).
- Optional `--force` (allows non-empty target).

## Outputs

- A pnpm workspace at `<target-dir>` with:
  - `packages/worker/` (always — the API)
  - `packages/web/` (if `--web`)
  - `packages/mobile/` (if `--mobile`)
  - `packages/shared/` (always)
  - `supabase/` (config + local stack)
  - `drizzle/` config + initial schema scaffolding
  - `wrangler.jsonc` with three envs
  - Top-level `package.json` with the named-scripts surface from `~/.claude/rules/09-infra.md`
  - `.claude/CLAUDE.md` (project pointer — one paragraph)
  - `.claude/hooks/` mirror of `~/.claude/hooks/` (for teammate inheritance)
  - `.claude/settings.json` mirror of `~/.claude/settings.json` hook wiring
  - `RUNBOOK.md` (deploy, rollback, key rotation, oncall)
  - `tsconfig.base.json`, `biome.json`, `.npmrc` (with `save-exact=true`), `.gitignore`
  - `initiatives/.gitkeep`, `docs/adr/.gitkeep`
- Status JSON: `.claude/scratch/scaffold-project-result-<sid>.json` with `{name, target_dir, packages: [...], next_steps: [...]}`.

## Procedure

1. **Confirm target dir empty** (or `--force`). Halt otherwise.
2. **Write package files** from templates at `~/.claude/skills/scaffold-project/references/`.
3. **`pnpm install`** with exact versions for each package (Context7 to confirm current stable major for each Tier-2 lib).
4. **Initialize git, create initial commit** `chore: scaffold <name>`.
5. **Surface next steps:**
   - Set Wrangler secrets per env.
   - `gh repo create` + push.
   - `supabase init` if customizing.
   - Create the first initiative with `/explore`.

## Template directory

`~/.claude/skills/scaffold-project/references/`:
- `package.json.template` (root + per-package)
- `wrangler.jsonc.template`
- `tsconfig.base.json.template`
- `biome.json.template`
- `supabase/config.toml.template`
- `RUNBOOK.md.template`
- `claude.CLAUDE.md.template`

(Generated by this skill; template files are populated from the `~/.claude/rules/*.md` declarations as the source of truth.)

## Failure mode

- Target dir non-empty without `--force` → halt.
- `pnpm install` failure → halt; surface stderr; do not retry destructively.
- Project name collides with an existing repo on the configured GitHub user → surface; let user resolve.
